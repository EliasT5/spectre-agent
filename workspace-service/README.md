# workspace-service

A standalone Node + TypeScript + Hono HTTP service (its own `workspace-service` Docker container) that hosts the Spectre Workspace backend in two modes behind one set of routes: **sandbox** — clone a GitHub repo into a slot under `WORKSPACE_ROOT`, edit/search/shell/run-tests, then finalize to push a branch and open a PR (up to 3 active slots, deletable); and **trusted folder** — real existing host folders bind-mounted into the container and registered via `WORKSPACE_TRUSTED_DIRS`, edited in place with no clone, never deletable, and finalize being an optional commit + push to the folder's current branch (no PR). Configure it with `WORKSPACE_SERVICE_TOKEN` (required shared auth secret; the service fails closed with 503 if it is unset), `WORKSPACE_ROOT` (sandbox slot root, default `/workspaces`), `WORKSPACE_TRUSTED_DIRS` (comma-separated absolute trusted paths), `GH_TOKEN` (server-side token injected into `git`/`gh` for clone/push/PR; never accepted from the client), and `PORT` (default `8010`). It is opt-in and reached only via the Spectre core/shell, which injects the `x-workspace-token` header (constant-time compared against `WORKSPACE_SERVICE_TOKEN`) on every proxied request; `GET /health` is the only unauthenticated route.
